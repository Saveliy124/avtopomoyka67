# 🚗 CarWash Management System

Профессиональная платформа для управления автомойкой (Бронирование, Касса, Админ-панель).

## 🛠 Стек технологий
- **Frontend**: React (Vite), TypeScript, Tailwind CSS, Zustand, React Query.
- **Backend**: Node.js, Express, PostgreSQL (pg).
- **Database**: PostgreSQL 16+.

---

## 🚀 Быстрый запуск (Инструкция для ИИ или Разработчика)

### 1. Предварительные требования
Убедитесь, что установлены:
- **Node.js** (v18+)
- **PostgreSQL 16** (Сервер должен быть запущен)

### 2. Настройка базы данных
Если PostgreSQL установлен локально:
1. Создайте базу данных `carwash_db`.
2. В файле `server/.env` проверьте `DATABASE_URL`. 
   *Пример: `postgres://postgres@localhost:5433/carwash_db` (пароль не требуется, если настроен trust в pg_hba.conf).*

### 3. Установка и инициализация (Backend)
Перейдите в папку сервера и выполните команды:
```bash
cd server
npm install
npm run db:init   # Создание таблиц
npm run db:seed   # Заполнение тестовыми данными
npm run dev       # Запуск бэкенда на http://localhost:4000
```

### 4. Установка и запуск (Frontend)
В корне проекта выполните:
```bash
npm install
npm run dev       # Запуск фронтенда на http://localhost:5173
```

## 📁 Структура проекта
- `/src` — Исходный код фронтенда (Vite/React).
- `/server` — Исходный код бэкенда (Node.js/Express).
  - `/server/src/sql/schema.sql` — Схема БД.
  - `/server/scripts/seed.js` — Начальные данные.
- `/server/pgdata` — (Опционально) Директория данных локального PostgreSQL.

## 📝 Заметки по миграции
- Бэкенд возвращает данные напрямую (без обертки `data`), что совместимо с текущим слоем API фронтенда.
- Все числовые поля (price, amount) в контроллерах принудительно приводятся к `Number`, чтобы избежать ошибок конкатекации строк в JS.
- **Итоговая стоимость**: Поле `price` в списке бронирований теперь автоматически включает стоимость всех выбранных дополнительных услуг (рассчитывается на сервере).
- **Отчетность**: Загруженность (`occupancy_rate`) теперь считается динамически как % занятых слотов от общего количества доступных в расписании на сегодня.
- **Управление**: Добавлен функционал удаления боксов (с проверкой на стороне БД).
- Порт БД по умолчанию в `.env` настроен на **5433**.
